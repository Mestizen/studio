Audit Log

Reconciliation Notes

Changes from the previous design system version, corrected against mestizen-studio.html as the single source of truth.

🔴
CORRECTED: --dark token value

Previous audit incorrectly documented --dark as #060626. The source HTML uses --dark: #1a1a2e. The old design-system.html also had #1a1a2e — both pre-existing files agree. #060626 never existed in the codebase. This document and tokens.css now use the correct value.

🔴
REMOVED: --charcoal token

--charcoal: #2d2d2d was listed in the previous design system but is never referenced anywhere in the source HTML or CSS. It has been removed from tokens.css and is not carried forward.

🟡
TOKENISED: Surface colors (previously hardcoded)

Three background values were hardcoded in the source and never in :root: #0f0f1e (services), #0a0a1a (work), #060612 (footer). They are now defined as --surface-services, --surface-work, --surface-footer in tokens.css. The source HTML should be updated to use these variables.

🟡
NOTED: nav.scrolled uses hardcoded rgba(26,26,46,0.9)

This is an alpha variant of --dark (#1a1a2e = rgb(26,26,46)). Correct, but should be updated to rgba(26,26,46,0.9) via a CSS variable for maintainability. Low priority.

🟢
CONFIRMED: All brand colors are consistent

--mint, --cream, --coral, --yellow, --purple, --purple-dark, --white are identical between the source HTML and the previous design system. No changes needed.

🟢
CONFIRMED: Typography stack is consistent

Bricolage Grotesque (display, weight 800) and Manrope (body) are consistent across all files. Font loading via Google Fonts CDN with the same weight range is confirmed.

🟡
SVG ASSETS: Inline duplicates found in source HTML

Asset 1, 3, 4, 5, 6 are duplicated as both standalone assets/*.svg files AND inline SVG code inside index.html. The inline versions use simplified/reimagined paths (not the exact same SVG). See SVG Assets section for full inventory. Recommendation: decide which version to keep as canonical, then reference via <img> or <use>.

🟡
Inline vs file: the two versions are NOT identical

The assets/*.svg files contain the final detailed Illustrator-exported paths. The inline SVGs in index.html are simplified hand-coded approximations used for decorative background elements. Recommendation: for any standalone usage (e.g. social, print, larger displays), use the assets/ files. For hero decorations, the inline version is intentionally lighter.
